# DivyOS Design System

## Anti-AI-Slop Directive
DivyOS must strictly avoid generic template aesthetics. It should not look like a standard Tailwind SaaS template, nor should it directly copy Windows, macOS, or GNOME. It must have its own unique, premium identity.

## Visual Language
- **Colors:** Curated, harmonious palettes. Avoid default web colors. Use deep, rich dark mode backgrounds (e.g., deep space blues, rich obsidians) and vibrant accent colors.
- **Typography:** Modern, highly legible fonts (e.g., Inter, Geist, or a custom variable font). Terminal needs a beautiful monospace font (e.g., JetBrains Mono, Fira Code).
- **Surfaces:** Glassmorphism used tastefully. Blurs, subtle borders, and gradient shadows to establish depth.
- **Animations:** Micro-interactions on hover, click, and state changes using Framer Motion. Spring animations for window movement to feel organic and snappy.

## UI Components
All standard UI components (buttons, inputs, dropdowns) will be custom-styled. We will strip out default Lovable/Radix styling that feels generic and replace it with our custom design language.

## Iconography
Every application and system element must have unique, recognizable iconography. Avoid overused generic icon packs where possible, or heavily customize them to fit the OS theme.
